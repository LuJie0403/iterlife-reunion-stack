# IterLife IDaaS 数据库基线说明

创建日期：2026-04-29  
最后更新：2026-04-30  
数据库版本：MySQL `8.0.35`

本文档仅用于说明当前 IDaaS 的数据库基线管理原则。

## 1. 当前原则

- `iterlife-idaas` 仓库不再保留本地 `.sql` 变更脚本。
- 当前生产数据库现状是 IDaaS 数据结构的唯一基准。
- 生产数据库已经具备后台自动多节点备份能力。
- 后续如需做数据库结构调整，应先基于生产库现状评估，再在变更窗口内执行。

## 2. 仓库口径

- 控制面仓不再维护 IDaaS 的人工执行 SQL 清单。
- 业务仓 `iterlife-idaas` 不再保留 `database/` 目录下的历史脚本文件。
- 正式设计文档只保留结构语义、业务规则和系统边界，不再引用具体脚本文件。

## 3. 执行要求

- 本地开发、测试和新环境初始化时，应以当前生产数据库结构为准进行对齐。
- 不应再以仓库中的历史 SQL 文件作为改库依据。
- 如需校验本地数据库是否满足当前后端运行要求，以应用启动期校验和当前生产库结构为准。
